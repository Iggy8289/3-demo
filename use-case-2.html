Research Automation

Building custom intelligence systems with AI assistance

The Situation

The Challenge: BD team needed funding opportunity intelligence but existing solutions were inadequate.

What We Had (External Consultant)

  • Cost: ~$5K/month ($60K annually)
  • Coverage: ~40 opportunities (transportation/mobility focus only)
  • Update frequency: Bi-weekly email reports
  • Technology scope: Limited to EV charging, transit electrification
  • Actionability: Generic summaries, required manual follow-up research

What We Needed

  • Full ESCO portfolio: EV, Solar, BESS, HVAC, Lighting, Building Envelope, Microgrids, etc.
  • Real-time updates: Know about opportunities when they're posted, not weeks later
  • Strategic intelligence: "Funding stacking" guidance (how to layer multiple programs)
  • Searchable/filterable: BD team can explore by client type, technology, deadline
  • Cost-effective: Better value than $60K/year

The Question: Could we build our own intelligence system rather than rely on external consultants?

Phase 1: Proof of Concept (Weekend 1)

Starting point: Used Claude Code (AI pair programming tool) to build initial web scraper

What I told Claude Code:

"I need to scrape the California Energy Commission funding opportunities page and extract: title, deadline, amount, status, and program type. Store results in CSV format."

What Claude Code did:

  • Created Python scraper in ~15 minutes
  • Handled HTML parsing and data extraction
  • Built CSV export functionality
  • Added error handling

First results: 10 CEC opportunities extracted successfully

Validation: "This actually works. Let's expand scope."

Phase 2: Expansion (Weekend 1-2)

Added 5 more sources with Claude Code:

  • California Air Resources Board (CARB)
  • PG&E business programs
  • Southern California Edison (SCE)
  • San Diego Gas & Electric (SDG&E)
  • Sacramento Municipal Utility District (SMUD)

Built "relevance scoring" system:

"Create a scoring algorithm (0-100) that ranks opportunities by relevance to OPTERRA's work. High scores for: K-12 schools, municipalities, EV+BESS+Solar combinations, CEC/CARB programs. Lower scores for: residential-only, federal grants requiring research focus."

Results after expansion:

  • 83+ opportunities tracked (vs ~40 previously)
  • Full ESCO technology coverage
  • Intelligent prioritization (60+ score = high priority)

Phase 3: Production Features (Week 2)

Added with Claude Code assistance:

  • Daily automation: Scheduled scraper runs, change detection
  • Email alerts: Notify team of new high-priority opportunities (60+ score)
  • Web dashboard: Searchable/filterable interface for BD team
  • "Funding stacking" intelligence: AI analyzes which programs can be layered (e.g., SGIP + Federal ITC + Utility incentives for solar+storage projects)
  • Data management: Version control, 365-day retention, error logging

What the Platform Looks Like

Dashboard View: 83 total opportunities, 81 active, 18 high priority (60+ relevance score), 33.3 average score. Real-time metrics updated daily.

Search & Filter Interface: Filter by project type (Transit Electrification, Solar+Storage, Building Efficiency, etc.), client type (K-12, Municipal, Transit Agency), budget range, technology, deadline. Smart matching shows why each opportunity scored as relevant.

Results Display: Priority scoring (96/100, 95/100, 90/100), match reasoning ("Strong K-12 focus, Solar+BESS combination, CEC program"), active status, one-click access to program details.
